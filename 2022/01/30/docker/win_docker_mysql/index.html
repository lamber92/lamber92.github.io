Docker-Desktop 部署MySQL

Lamber

2022-01-30

本篇故意尽量使用UI界面交互替代docker命令，达到最终部署目的。

1. 拉取镜像

  • 搜索关键字，选择官方镜像，选择最新版本 (本地调试使用，因此使用最新版)：

2. 创建容器

  • 找到刚刚拉取完成的镜像，点击运行按钮

  • 接下来是填写容器名称、环境变量

    • 可以先对照dockerhub上对应镜像说明中的配置信息，选择自己需要的填
    • 镜像说明地址：https://hub.docker.com/_/mysql

    • 按需填写这几项，然后点击”Run”按钮。

  • 查看容器日志

3. 连接测试

上面配置已经将3306端口映射到宿主机，在宿主机用Navicat连一下就行，不赘述。

4. 绑定自定义网络

这一步是为了能将Mysql的端口映射到docker子网内，给其他需要依赖Mysql的服务提供一个稳定的ip地址，并且形成网络隔离

这个没办法在Docker-Desktop上操作，打开终端

  • 先查看当前有哪些网络

  • 看到有 bridge 属性的网络，看看哪些容器已经绑定到该网络上

    • 通过这个信息，已经确认 mysql_local_dev 已经绑定到 bridge 中，无需再另外创建；
    • mysql 在 bridge 中的ip地址是 172.17.0.2 。
